About This System

This project is a Secure Phishing Detection System designed as an academic prototype. It demonstrates how phishing indicators can be detected using an explainable, rule-based approach, supported by secure authentication and audit logging.

Rule-Based Detection

The detector analyses submitted email text and checks for common phishing indicators such as:

  • Urgency / threat language (e.g., “urgent”, “immediately”)
  • Credential requests (e.g., “verify account”, “password”, “sign in”)
  • Financial pressure terms (e.g., “payment”, “invoice”, “refund”)
  • Suspicious URLs (IP-based links, punycode, risky TLDs)
  • Attachment and macro-related tricks

Explainable Results

Instead of returning only a label, the system provides:

  • Classification (LEGIT / SUSPICIOUS / PHISHING)
  • Risk score based on weighted rules
  • Triggered reasons showing which indicators matched

This improves transparency and helps users understand why the content is suspicious.

Security Features

  • Secure login with salted password hashing (PBKDF2)
  • Session-based access control for authenticated features
  • Role-based access (User vs Admin)
  • Audit logging for login, logout, detection and admin access events
  • History storage so users can review past detections

Limitations

  • This prototype uses mock email samples (no real inbox integration).
  • Rule-based detection may produce false positives or miss new attack styles.
  • Future improvements can include ML models and threat intelligence APIs.

How to Use

  1. Register an account and log in.
  2. Paste email content into the dashboard and click Detect.
  3. Review the classification, score, and explainable reasons.
  4. Check History for past results. Admin users can review logs in the Admin panel.
